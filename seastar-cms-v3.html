설명서

작업 순서와 라우팅 기준

기본 작업 순서
  • `케이블` 버튼으로 SCH 파일을 불러옵니다.
  • `노드` 버튼으로 NODE 파일을 불러옵니다.
  • 케이블 리스트를 더블클릭해 상단 편집기에서 값을 수정합니다.
  • `저장+재산출` 또는 `전체경로`로 경로를 다시 산출합니다.
  • `전체검증`으로 그래프, 경로, 맵 일치 여부를 확인합니다.
  • 완료 후 `엑셀` 또는 `서버저장`을 사용합니다.
라우팅 규칙
  • 총 길이는 `GRAPH_LENGTH + FROM_REST + TO_REST`로 계산됩니다.
  • `CHECK_NODE`를 입력하면 입력 순서대로 반드시 경유합니다.
  • `FROM_NODE`와 `TO_NODE`가 같으면 해당 노드 `linkLength`를 기본 길이로 사용합니다.
  • 연속 노드쌍의 relation edge가 실제로 있어야 검증 PASS가 가능합니다.
  • 좌표가 없는 노드는 맵 상태에서 경고가 표시됩니다.
화면 사용 기준
  • 주 작업 화면은 `케이블` 탭입니다. 리스트를 최대한 많이 보도록 구성했습니다.
  • 우측 패널은 선택 케이블의 길이 분해와 검증 결과만 빠르게 확인하는 용도입니다.
  • 상단 편집기에는 자주 바꾸는 항목만 먼저 보이고, 추가 SCH 필드는 접어서 관리합니다.
  • `경로` 탭은 수동 경로 미리보기와 `CHECK_NODE` 검토용입니다.
로그인 / 저장
  • 관리자 로그인, Google, Naver는 서버 인증 설정이 되어 있어야 동작합니다.
  • 서버 저장은 현재 프로젝트 스냅샷을 저장합니다.
  • 엑셀 반출 전에는 검증 상태와 총 길이를 반드시 확인하세요.
